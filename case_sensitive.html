Case Sensitivity in SQL:


SQL is typically not case-sensitive for keywords. This means that SQL statements like SELECT, FROM, WHERE, and others can be written in uppercase or lowercase, and they will be recognized by the database engine.

Example: Copy
-- Keywords with UPPER case Letters
SELECT product_name, price FROM products ORDER BY price DESC;

-- Keywords with LOWER case Letters
select product_name, price from products order by price dec;

--The above two queries will return the same result, because all the SQL keywords are case-insensitive.

In some databases, such as MySQL and PostgreSQL, identifiers are case-insensitive,
while in others, such as Oracle and Microsoft SQL Server, identifiers are case-sensitive. This means that the names of tables, columns, and other database objects must be spelled exactly the same, including capitalization, in order to be referenced correctly.

It is advised to refer to the specific database's documentation to understand its behavior regarding case sensitivity.


There are a few things to keep in mind when dealing with case sensitivity in SQL:

  • When creating database objects, it is generally best to use all lowercase letters for names. This will avoid any potential problems with case sensitivity, regardless of the database that you are using.
  • If you are working with a database that has case-sensitive identifiers, you must use double quotes around all object names in your SQL statements. This will tell the database to treat the name as a case-sensitive identifier.
  • When using the LIKE operator, you can use the % wildcard character to match any number of characters, including spaces and case. This can be useful for performing case-insensitive searches.
