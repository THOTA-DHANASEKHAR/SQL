SQL Data Types


SQL supports various data types that define the kind of data that can be stored in a database table's columns. These data types help maintain data integrity and improve query performance. Here are some common SQL data types:


Numeric Data Types:

INTEGER (INT): Represents whole numbers (e.g., -10, 0, 42).

SMALLINT: Stores small whole numbers, often with a smaller range than INTEGER.

BIGINT: Stores large whole numbers with a larger range than INTEGER.

NUMERIC(P, S) or DECIMAL(P, S): Represents exact numeric values with a specified precision (P) and scale (S). P is the total number of digits, and S is the number of digits to the right of the decimal point.

FLOAT(P): Represents approximate numeric values with a specified precision (P). It's suitable for scientific and engineering calculations.

REAL: A single-precision floating-point number.

DOUBLE PRECISION: A double-precision floating-point number.


Character String Data Types:

CHAR(N): Stores fixed-length character strings with a specified length (N). Padded with spaces to reach the specified length.

VARCHAR(N): Stores variable-length character strings with a maximum length of N characters.

TEXT: Stores large text data.


Date and Time Data Types:

DATE: Stores dates (year, month, day).

TIME: Stores times (hours, minutes, seconds).

TIMESTAMP: Stores date and time together.

INTERVAL: Represents a period of time.


Binary Data Types:

BINARY(N): Stores fixed-length binary data with a specified length (N).

VARBINARY(N): Stores variable-length binary data with a maximum length of N bytes.

BLOB: Stores large binary objects, such as images, audio, or binary documents.


Boolean Data Type:

BOOLEAN (BOOL): Represents true or false values.


Enumerated Data Types:

ENUM: Stores one of a predefined list of values.


Bit Data Types:

BIT(N): Stores fixed-length bit strings with a specified length (N).


Geospatial Data Types:

GEOMETRY: Stores geometric data, such as points, lines, and polygons.


JSON Data Types:

JSON: Stores JSON-formatted data.


Custom Data Types:

Some database systems allow you to define custom data types, which can be based on existing types but have additional constraints or behaviors.


Special Data Types:

XML: Some database systems support XML data types for storing XML documents.

These data types may vary slightly from one database management system (DBMS) to another (e.g., MySQL, PostgreSQL, SQL Server, Oracle). Additionally, DBMSs may have their own unique data types and extensions. It's essential to refer to the documentation of the specific DBMS you are using for detailed information on data types and their usage in that system.
