Overview of SQL Commands


SQL is a powerful tool for interacting with relational databases. It enables users to perform various operations on the data stored within a database. Here are the main types of SQL statements:


Data Definition Language (DDL)


  • CREATE: Creates a new database object, such as a databese, table, view or an index with specified names, columns and data types.

  • ALTER: Modifies the structure of an existing database object such as adding, modifying, or deleting columns.

  • DROP: Deletes a database object, such as a table or an index.

  • TRUNCATE: Removes all rows from a table while keeping its structure intact.

  • COMMENT: Adds comments to database objects, helping with documentation.

Data Query Language (DQL)


  • SELECT: Retrieves data from one or more tables based on specified criteria.

  • FROM: Specifies the table(s) from which to retrieve data.

  • WHERE: Filters data based on specified conditions.

  • GROUP BY: Groups data based on one or more columns.

  • HAVING: Filters grouped data based on aggregate conditions.

  • ORDER BY: Sorts the result set based on specified columns.

  • JOIN: Combines rows from multiple tables based on related columns.

Data Manipulation Language (DML)


  • INSERT: Adds new rows of data into a table.

  • UPDATE: Modifies existing data in a table based on specified conditions.

  • DELETE: Removes rows of data from a table based on specified conditions.

  • MERGE: Combines INSERT, UPDATE, and DELETE operations into a single statement (not supported in all database systems).

  • SELECT INTO: The SELECT INTO statement copies data from one table into a new table

Data Control Language (DCL)


  • GRANT: Assigns specific privileges to users or roles.

  • REVOKE: Removes specific privileges from users or roles.

Transaction Control Language (TCL)


  • COMMIT: Saves all changes made during the current transaction.

  • ROLLBACK: Undoes changes made during the current transaction and restores the data to its previous state.

  • SAVEPOINT: Creates a point within a transaction to which you can later roll back.

  • SET TRANSACTION: Sets properties for a transaction, such as isolation level and access mode.
