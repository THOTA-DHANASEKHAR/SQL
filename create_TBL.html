SQL_Syntax

CREATE TABLE

To create a SQL table, you need to use the CREATE TABLE statement. This statement creates a new database with the specified name.

Table:

  • A table in SQL is a structured collection of data organized into rows and columns. Each row in a table represents a record or an entry, while each column represents a specific attribute or field of that record. Tables are used to store, manage, and retrieve data.
  • Tables have a predefined structure defined by the columns they contain. Each column has a name and a data type that specifies the type of data it can hold (e.g., INT, VARCHAR, DATE).
  • Tables can have a primary key, which is a unique identifier for each row, and they can also have relationships with other tables through foreign keys.
  • SQL operations like INSERT, SELECT, UPDATE, and DELETE are used to manipulate data in tables.

Syntax for Table:

Example: Copy 
CREATE TABLE table_name (
column1 datatype,
column2 datatype,
... );

Example Query for Table:

Example: Copy 
CREATE TABLE employees (
employee_id INT PRIMARY KEY,
first_name VARCHAR(50),
last_name VARCHAR(50),
hire_date DATE );

This will create a new table with four columns: employee_id, first_name, last_name, and hire_date. The employee_id column is the primary key.

Copy table with Data:

CREATE TABLE statement with a SELECT statement to copy data from the source table into the new table.

Example: Copy 
CREATE TABLE employees_new
AS
SELECT *
FROM employees;

This will copy all of the data from the customers table to the customers_new table.


Copy table Schema:

To create a table without copying data, you can use the CREATE TABLE AS statement. This statement creates a new table with the same structure as an existing table, but without any of the data.

Example: Copy 
CREATE TABLE employees_new
AS (
SELECT *
FROM employees
WHERE 1=2
);

The WHERE 1=2 clause ensures that no rows are returned from the employees table, and therefore no data is copied to the new table.


Table Naming:


1. Use Singular Nouns: Table names should generally be in the singular form, such as "employee" instead of "employees." This helps with readability.

2. Use Descriptive Names: Choose names that reflect the type of data the table stores. For example, "order_details" is more descriptive than "table1."

3. Avoid Spaces and Special Characters: Stick to alphanumeric characters and underscores. Avoid spaces, hyphens, and special characters in table names.

4. Use All Lowercase: Consistently using lowercase letters for table names is a common convention.

5. Use Underscores for Compound Words: If a table name consists of multiple words, separate them with underscores, e.g., "customer_orders" or "product_categories."

6. Use prefixes and suffixes to group related tables together: For example, you could use the prefix products_ for all tables that store product information, and the suffix _history for all tables that store historical data.

Column Naming:


1. Use Descriptive Names: Columns should have names that clearly indicate the type of data they store. For example, use "first_name" instead of "fld1."

2. Use All Lowercase: Consistently use lowercase letters for column names.

3. Avoid Spaces and Special Characters: Stick to alphanumeric characters and underscores for column names.

4. Be Consistent: Maintain consistency in naming conventions across your database. If you use underscores in one table or column name, use them in others as well.
